Generator

This part will be executed before parsing request input with the signature for generate variation request base on the original request. Generator inly avaliable in type: fuzz signature.

Generator

API Generator Description Example
Method Gen request based on method Method("PUT") , Method()
Query Gen request based on Query String Query("{{.original}}{{.payload}}"), Query("{{.original}}{{.payload}}", "new")
Path Gen request based on Path of URL Path("{{.payload}}", "ext")
Body Gen request based on Request Body Body("{{.original}}{{.payload}}")
Headers Gen request based on Request Header Header("{{.original}}{{.payload}}", "X-Sample")

Using Generator

id: sqli-01-01
info:
  name: SQL Injection Error based
  risk: Critical
type: fuzz

payloads:
  - '''"'
  - '`}'

requests:
  - generators:
      - Query("{{.payload}}")
    detections:
      - >-
        StringSearch("response", "mysql_fetch_array")

If you want to generate single request with many generator do like this - Query("{{.payload}}"); Method("PUT").

Method

Method() will generate "GET", "POST", "PUT", "HEAD", "PATCH" based on request input. If parameter specified Method("PUT") only one request with *PUT method generated.

Query, Body and Header

Query("{{.original}}{{.payload}}") will generate number of based on number of query string in URL of the request input.

For example with signature sqli-01-01 and our URL is http://example.com/?q=123. We gonna get two new requests with URL http://example.com/?q=123'" and http://example.com/?q=123%60}.

If we use Query("{{.original}}{{.payload}}","new") generator then we gonna get http://example.com/?q=123&new='".

Header() and Body() have same usage of Query() generator.

For example. Body("{{.original}}{{.payload}}") generate number of based on number of request body parameter of the request input.

Header("{{.original}} \ {{.payload}}", "X-Command") will generate a new request with X-Command header with our payload.

Path

First of all, Jaeles split path of URL like this

URL: http://example.com/rest/products/6/reviews?q=123
Path: => /rest/products/6/reviews
         0   1        2  3     -1

and have some special parameter ext, last, 0/1/2 and *.

For example our payload is .json.

Path("{{.payload}}", "last") mean end of the Path will be replace by the payload to this /rest/products/6/reviews.json.

Path("{{.payload}}", "ext") mean extension of the Path (like .php) but our URL doesn’t have extension.

Path("{{.payload}}", "1") mean position 1 of the Path will be replace by the payload to this /rest.json/products/6/reviews.

Path("{{.payload}}", "0/2/3") mean position 0,2,3 of the Path will be replace by the payload to this /.json/rest/products/6/reviews, /rest/products.json/6/reviews and /rest.json/products/6.json/reviews

and Finally Path("{{.payload}}", "*") will generate payload gonna replace every single path. If you don’t specified position parameter like this Path("{{.payload}}"), by default it’s equal to Path("{{.payload}}", "last").
